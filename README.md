# Pipelock GitHub Action

Run [Pipelock](https://github.com/luckyPipewrench/pipelock) security scans in your CI pipeline.

Three scan modes:
- **audit** - Scan a project for security risks, detect agent types, find secrets
- **git-scan-diff** - Scan git diffs for leaked secrets before they reach main
- **integrity-check** - Verify workspace files haven't been tampered with

## Usage

### Audit a project

```yaml
- uses: actions/checkout@v4

- uses: luckyPipewrench/pipelock-action@v1
  with:
    mode: audit
    directory: '.'
```

### Scan diffs for secrets

**Important:** Use `fetch-depth: 2` (or more) so the diff has a parent commit to compare against.

```yaml
- uses: actions/checkout@v4
  with:
    fetch-depth: 2  # Required for diff scanning

- uses: luckyPipewrench/pipelock-action@v1
  with:
    mode: git-scan-diff
```

### Check workspace integrity

Requires a `.integrity-manifest.json` generated by `pipelock integrity init`.

```yaml
- uses: actions/checkout@v4

- uses: luckyPipewrench/pipelock-action@v1
  with:
    mode: integrity-check
    directory: '.'
    manifest: '.integrity-manifest.json'
```

## Inputs

| Input | Required | Default | Description |
|-------|----------|---------|-------------|
| `mode` | Yes | | Scan mode: `audit`, `git-scan-diff`, `integrity-check` |
| `version` | No | `latest` | Pipelock version to install |
| `config` | No | | Path to `pipelock.yaml` config file (git-scan-diff mode only) |
| `directory` | No | `.` | Directory to scan (audit and integrity modes) |
| `manifest` | No | | Integrity manifest path |
| `fail-on-findings` | No | `true` | Fail the job if findings are detected |

## Outputs

| Output | Description |
|--------|-------------|
| `findings` | JSON output from the scan |
| `exit-code` | Pipelock exit code (0 = clean, 1 = findings) |

## Job Summary

Each mode writes a markdown summary to the GitHub Actions job summary:

- **audit**: Score table, severity counts, critical findings list
- **git-scan-diff**: Findings table with file, line, pattern, severity
- **integrity-check**: Pass/fail with violation details

## Examples

See [`examples/pipelock-ci.yml`](examples/pipelock-ci.yml) for a complete workflow.

### Use findings in later steps

```yaml
- uses: luckyPipewrench/pipelock-action@v1
  id: pipelock
  with:
    mode: audit
    fail-on-findings: 'false'

- run: |
    echo "Exit code: ${{ steps.pipelock.outputs.exit-code }}"
    echo '${{ steps.pipelock.outputs.findings }}' | jq '.score'
```

### Pin to a specific version

```yaml
- uses: luckyPipewrench/pipelock-action@v1
  with:
    mode: audit
    version: 'v0.1.5'
```

## Security

- The action downloads Pipelock binaries from GitHub Releases and verifies SHA256 checksums
- All inputs are passed via environment variables to prevent script injection
- Pin to a specific version in production workflows for reproducibility
